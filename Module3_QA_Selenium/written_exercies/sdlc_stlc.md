# Task 1: Software Development Life Cycle (SDLC)

The Software Development Life Cycle (SDLC) is a structured process used to design, develop, test, deploy, and maintain software applications.

## SDLC Phases

1. Requirement Gathering
2. Planning
3. System Design
4. Development
5. Testing
6. Deployment
7. Maintenance

---

# Task 2: Software Testing Life Cycle (STLC)

The Software Testing Life Cycle (STLC) defines the activities performed during software testing.

## STLC Phases

1. Requirement Analysis
2. Test Planning
3. Test Case Development
4. Test Environment Setup
5. Test Execution
6. Defect Reporting
7. Test Closure

---

# Task 3: SDLC vs STLC

| SDLC | STLC |
|------|------|
| Focuses on software development | Focuses on software testing |
| Performed by developers | Performed by QA/Testers |
| Ends with deployment | Ends with test closure |
| Produces software | Ensures software quality |

---

# Task 4: V-Model

The V-Model maps every development phase to a corresponding testing phase.

| Development Phase | Testing Phase |
|-------------------|---------------|
| Requirements | Acceptance Testing |
| System Design | System Testing |
| Architecture Design | Integration Testing |
| Module Design | Unit Testing |

## Advantages

- Early defect detection
- Better quality
- Easy traceability
- Suitable for stable requirements

## Disadvantages

- Not flexible for changing requirements
- Difficult for Agile projects

---

# Task 5: Agile Testing

Agile Testing is performed continuously throughout the software development process.

### Characteristics

- Continuous Testing
- Continuous Feedback
- Customer Collaboration
- Frequent Releases
- Sprint-based Testing

### Advantages

- Faster delivery
- Early bug detection
- Improved customer satisfaction

---

# Task 6: Shift-Left Testing

Shift-Left Testing means testing starts as early as possible during software development.

### Benefits

- Finds defects early
- Reduces development cost
- Improves software quality
- Reduces project risk

# Task 7: Example

### Scenario

A new "Student Registration" feature is being developed.

### SDLC

- Requirement collection
- Design database
- Develop APIs
- Test feature
- Deploy application

### STLC

- Analyze requirements
- Prepare test cases
- Execute tests
- Report defects
- Verify fixes
- Close testing

---

# Conclusion

Using SDLC and STLC together helps deliver reliable, high-quality software. Agile and Shift-Left practices improve development speed while reducing defects through continuous testing.
